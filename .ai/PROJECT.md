# AIへの指示

* このファイルを、ブログ記事作成時の唯一の管理ドキュメントとして扱うこと。
* 記事を作成する前に必ずこのファイルを確認し、記載されたルールに従うこと。
* 記事ごとの内容や進捗はこのファイルに記載しない。
* ブログ運営全体に関わるルールのみを管理すること。
* 更新履歴はGit logで管理し、このファイルには記載しない。

# ブログ概要

* **サイト:** GitHub Pages（Jekyll）
* **目的:** ゲームボーイホームブリュー開発とAIを活用した開発手法を記録・公開する。
* **対象読者:** ゲームボーイ開発やAI支援開発に興味のある人。
* **基本方針:** 実際に行った作業を継続的に記録し、後から見返しても開発の流れが分かる技術ブログとする。

# 記事の基本方針

* 実際に行った作業のみを書く。
* 未実装の内容や推測は事実と区別して記載する。
* 問題が発生した場合は、原因と解決方法を記載する。
* AIを利用した箇所も隠さず記載する。
* 誇張した表現は避け、技術ブログとして事実を中心にまとめる。
* 初心者が再現できることを意識して執筆する。

# 記事の単位

* 原則として、開発プロジェクトのPROJECT.mdに記載したWBSの完了単位を1記事とする。
* 小規模なWBSは複数まとめて1記事としてもよい。
* 大規模なWBSは複数記事へ分割してもよい。
* 記事は時系列に沿って作成する。

# 記事構成

基本的に以下の構成で作成する。

1. はじめに
2. 今回実施した内容
3. 実装・設定内容
4. 苦労した点・ハマった点
5. 今後の予定
6. まとめ

内容に応じて見出しは追加・変更してもよい。

# 文体

* 「です・ます調」を使用する。
* 一文を長くしすぎない。
* 技術用語は必要に応じて簡単に補足する。
* コード掲載は必要最小限とする。
* 長いコードは掲載せず、概要を説明する。
* 箇条書きを適切に使用し、読みやすさを優先する。

# Jekyllルール

## Front Matter

すべての記事にFront Matterを付与する。

### date

Jekyllの `date` は日付だけでなく時刻まで記載する。

同じ日に複数の記事を作成することがあるため、記事作成順に並ぶよう時刻を設定する。

タイムゾーンは `+0900` とする。

例

```yaml
date: 2026-06-27 20:30:00 +0900
```

## カテゴリ

既存サイトで使用しているカテゴリを利用する。

## タグ

* すべて小文字
* 単語区切りはハイフン（`-`）
* `gameboy` は分割しない

例

```text
gameboy
rgbds
assembly
minesweeper
ai
codex
hugetracker
jekyll
github-pages
```

## 画像

画像は手動で作成・配置する。

AIやCodexは、記事内で使用する画像ファイル名と配置先を指示・参照してよいが、画像ファイル自体の作成・移動・配置は原則として行わない。

画像は以下へ配置する。

```text
/assets/images/YYYY-MM-DD-記事スラッグ/
```

記事内では

```liquid
{{ site.baseurl }}/assets/images/...
```

を使用する。

画像ファイル名は内容が分かる名前を付ける。

例

```text
cursor_before.png
cursor_after.png
input_processing.png
```

記事作成時に画像がまだ配置されていない場合は、記事内の画像パスのみ記述し、画像配置は手動作業として残す。

# AI利用時のルール

* ChatGPTやCodexを利用した場合は、必要に応じてその内容も紹介する。
* AIが生成した内容はそのまま掲載せず、実際に確認した内容のみを書く。
* AIとの試行錯誤も有益であれば紹介する。

# 記事作成チェックリスト

記事作成時は以下を確認する。

* Front Matterが設定されている
* タイトルが内容を表している
* タグ・カテゴリがルールに従っている
* 画像パスが `site.baseurl` を使用している
* スクリーンショットを適切に配置している
* 内容が実際の作業と一致している
* 次回予定を簡潔に記載している
* 誤字・脱字を確認している
* Jekyllで表示確認を行っている
