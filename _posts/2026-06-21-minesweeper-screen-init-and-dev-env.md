## はじめに

今回は、ゲームボーイ向けマインスイーパー開発の続きとして、タイル画像の作成と画面初期化を進めました。

前回までで最小ROMの作成やビルド環境の整備はできていたので、今回は実際にゲーム画面らしいものを表示するところまで進めます。

## タイル画像を作成する

まずはマインスイーパーで使用するタイル画像を作成しました。

用意したタイルは以下の15種類です。

|番号|内容|
|---:|---|
|0|未開封|
|1|開封済み(0)|
|2|開封済み(1)|
|3|開封済み(2)|
|4|開封済み(3)|
|5|開封済み(4)|
|6|開封済み(5)|
|7|開封済み(6)|
|8|開封済み(7)|
|9|開封済み(8)|
|10|フラグ|
|11|地雷|
|12|爆発した地雷|
|13|誤ったフラグ|
|14|カーソル|

当初は「地雷」と「爆発した地雷」だけでよいかとも思いましたが、ゲームオーバー時に何が起きたのかを分かりやすくするため、誤ったフラグも追加しました。

これにより、ゲーム終了時に次のような表示ができます。

- 正解の地雷は地雷タイルで表示
- 踏んだ地雷は爆発した地雷タイルで表示
- 間違って置いたフラグは誤ったフラグタイルで表示

マインスイーパーでは、ゲームオーバー後に「どこが間違っていたか」が分かることも大事なので、この構成にしました。

## AIでドット絵を作るのはまだ難しい

途中で、AIに32×32ドットのモンスター画像を作らせる実験もしました。

ゴブリンのようなキャラクターを文字列表現で作れるか試してみましたが、実際にはかなり難しいことが分かりました。

特に問題だったのは以下です。

- 1ドット単位の形が安定しない
- 左右対称が崩れやすい
- シルエットが弱い
- 画像として見たときに意図した形から離れやすい

そのため、現時点ではAIに完成品のドット絵を作らせるよりも、AIはデザイン案やアイデア出しに使い、最終的なドット絵は人が作成する方が現実的だと感じました。

今回はドット絵エディタの **Edge** を使ってタイル画像を作成しました。

ブラウザのMicrosoft Edgeと名前が同じなので少し紛らわしいですが、ここでいうEdgeはドット絵エディタの方です。

## 画像仕様を決める

タイル画像やフォント画像は、ゲームボーイ向けに4色で統一します。

今回、PNG画像のパレットは以下の4色に決めました。

|Index|Color|
|---:|---|
|0|#E0F8D0|
|1|#88C070|
|2|#346856|
|3|#081820|

`rgbgfx` で変換するときも、パレットを明示的に指定します。

```bash
rgbgfx -c "#E0F8D0,#88C070,#346856,#081820" -o $@ $<
```

パレットを明示しておくことで、画像編集ソフトやPythonツールで生成したPNGを安定して2bppへ変換できるようになります。

## フォント画像の再生成

ステータスバーには次のような文字列を表示します。

```text
MINE:010 TIME:000
```

ここで `:` が必要になりました。

既存の `font.png` には `:` が含まれていなかったため、フォント画像の作成方法を見直すことにしました。

フォントには Nuきなこもち を使用しているので、OTFファイルからPythonで `font.png` を生成するスクリプトを作成しました。

```text
assets/NuKinakoMochi-Reg.otf
```

を入力として、

```text
assets/font.png
```

を出力します。

PythonではPillowを使います。

```bash
python tools/generate_font_png.py
```

これに合わせて、Python開発環境も整備しました。

- `.venv` を使用する
- `requirements.txt` で依存ライブラリを管理する
- Pythonツールは `tools/` に配置する
- 使い方は `tools/README.md` に記載する

ゲーム本体はRGBDSで開発しますが、フォント生成や画像変換などの支援ツールにはPythonを使う方針です。

## 画面初期化を実装する

次に、WBSの「画面初期化」を進めました。

今回の完了条件は、起動時に次の内容を表示することです。

- 1行目にステータスバーを表示する
- 9×9の盤面を未開封タイルで表示する
- WindowやSpriteはまだ使わない
- 入力処理やカーソル移動もまだ実装しない

盤面はBG座標で `(5, 4)` から表示するようにしました。

BG座標はタイル単位なので、ピクセルでいうと `(40, 32)` の位置です。

```text
MINE:010 TIME:000
```

を上部に表示し、その下に9×9の未開封マスを並べました。

![画面初期化]({{ site.baseurl }}/assets/images/2026-06-21-minesweeper-screen-init/screen-init.png)

これで、ようやくゲーム画面らしい表示になりました。

## VS Codeのincludeパス設定

ビルドは問題なく通っていましたが、VS Code上では `INCLUDE` のパス解決ができず、エラー表示が出ていました。

RGBDS拡張では、includeパスをワークスペース相対で指定する必要がありました。

```json
{
  "rgbdsz80.includePath": [
    "include",
    "src"
  ]
}
```

`${workspaceFolder}/include` のように書くと、この拡張ではうまく動作しませんでした。

`.vscode` ディレクトリは `.gitignore` に入っているため、設定ファイルはGit管理しない運用にしています。
